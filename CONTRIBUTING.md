# Contributing to Spark Graph Viewer

First off, thank you for considering contributing to ***Spark Graph Viewer***! It's people like you who make open-source projects better.

As this is a personal project, your help is highly appreciated, and I will personally review every submission.

## 🐛 How to Report Bugs
If you find a bug, please help by opening an issue:

   1. Check existing issues to see if the bug has already been reported.
   2. Use a clear title that describes the problem.
   3. Describe the steps to reproduce the issue, including your environment (Docker version, Spark version, etc.).
   4. Attach screenshots or logs if possible.

## 💡 Feature Requests
Have an idea to make ***Spark Graph Viewer better***? Feel free to open an issue with the tag enhancement. Explain why the feature would be useful and how you imagine it working.

## 🚀 Pull Request Process
I welcome your contributions! To ensure a smooth review, please follow these steps:

   1. Fork the repository and create your branch from main.
   2. Keep it focused: A single PR should address one bug fix or one new feature.
   3. Test your changes: Ensure the Docker environment builds correctly and the Spark processing remains stable.
   4. Self-review: Check your code for comments, debugging logs, or unnecessary changes.
   5. Submit the PR:
   * Provide a clear description of what you changed.
      * Reference any related issues (e.g., Closes #12).
   6. Wait for review: Since this is a personal project, I will evaluate each PR as soon as possible. I might ask for changes or clarifications before merging.

## 🎨 Style Guidelines

* Javascript: Keep it clean and follow D3.js best practices for data binding.
* Python/PySpark: Follow PEP 8 standards.
* CSS: Use Bootstrap classes whenever possible to maintain UI consistency.
